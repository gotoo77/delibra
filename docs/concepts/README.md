# Delibra Concept Notes

Concept notes are the official parking place for useful ideas that are not core.

They let Delibra document pressure on an abstraction without admitting it into the durable model too early.

Concept notes are not a backlog for interesting ideas.

Concept notes document tensions, not ideas.

A tension is evidence that the current model cannot absorb something cleanly.

A tension may come from a real protocol, implementation pressure, or architectural review.

## Status Ladder

### Idea

The concept has been named, but no serious validation exists yet.

### Hypothesis

The concept may represent a real abstraction, but it has not been proven across unrelated protocols.

### Candidate

The concept has pressure from at least three unrelated protocols, cannot reasonably live outside the core, and the current model is becoming less coherent without it.

### Core

The concept has been accepted into the durable core by an ADR.

## Required Fields

Each concept note should record:

- Status;
- Core status;
- Motivation;
- Evidence;
- Protocols requiring it;
- Whether it can remain outside the core;
- Decision.

## Rule

Important abstractions are not chosen. They become inevitable.

Concepts should remain boring before powerful.
